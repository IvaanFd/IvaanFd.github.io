Deployment of a Landing Zone under a Hub-and-Spoke Architecture in AWS

This project focuses on the design and deployment of a Landing Zone in AWS, utilizing a Hub-and-Spoke Architecture to establish a secure and scalable cloud infrastructure. The Landing Zone was designed to facilitate centralized resource management and structured organization, supporting both development and production environments.

The primary goal of the project is to create an efficient and automated infrastructure configuration that accelerates deployment cycles, reduces configuration errors, and improves operational reliability. The project includes an initial manual setup phase to familiarize with AWS services, followed by full automation using Infrastructure as Code (IaC) tools like Terraform and Bash scripting.

Infrastructure Overview

High Level Design

Explanation

The infrastructure is designed around three distinct environments:

  • Production: A dedicated environment with isolated VPCs for running live workloads.
  • Development: A mirrored environment to Production for testing and development purposes, with CIDR blocks adjusted to avoid overlap.
  • Shared: A centralized VPC that hosts common resources such as AWS service endpoints, DNS configurations, and other shared services accessible by both Production and Development environments.

Network Architecture

The Hub-and-Spoke architecture ensures each VPC has a clear role:

  • Shared VPC: Contains AWS endpoints (e.g., S3, DynamoDB), which are shared across other VPCs.
  • Hub VPC: Acts as the core network connection point, providing routing to Spoke VPCs.
  • Spoke VPCs: Include subnets for Front, Back, and Data tiers, each hosting specific EC2 instances or services.

Purpose and Methodology

The purpose of this project is to establish a modular and replicable cloud infrastructure that can be easily managed and scaled across different environments. The methodology used for deployment includes two main milestones:

  1. Manual Deployment Phase: Designed to become familiar with AWS services and manually configure key resources such as VPCs, subnets, and peering connections.
  2. Automated Deployment Phase: Terraform was used as an Infrastructure as Code (IaC) tool to automate provisioning processes, complemented by Bash scripting to optimize workflows.

The main components of the Landing Zone include:

  • Shared VPC: Hosts shared services such as AWS endpoints and private DNS.
  • Hub VPC: Centralized environment for managing shared services and connecting to Spoke VPCs.
  • Spoke VPCs: Isolated environments designed to host development and production workloads.
  • Route tables: Configured to direct traffic between VPCs via peering connections while maintaining strict control over the allowed routes to ensure security and isolation.
  • VPC Peering: Establishes secure interconnection between the Hub and Spokes, enabling communication while ensuring no unnecessary traffic flows between unrelated VPCs.
  • Endpoints: Deployed in the Shared VPC and shared across all Spoke VPCs, allowing efficient access to services like S3 and EC2 without traversing the public internet.
  • NACLs and Security Groups: Configured with reusable, centralized rules to manage inbound and outbound traffic across subnets and instances, reducing redundancy while enforcing strict security controls.
  • Secure Parameters: Managed in AWS Systems Manager Parameter Store as encrypted SecureString values, ensuring sensitive data such as SSH keys and credentials are securely stored and accessed programmatically.

Key Features of This Deployment

  • Modular Design: Allows the addition of new VPCs or subnets easily to scale based on organizational needs.
  • Automation: Terraform ensures consistency and reduces human error in infrastructure configuration.
  • Security and Governance: Implements AWS best security practices, including Network ACLs, Security Groups, and private hosted zones.
  • Efficiency: Bash scripts optimize Terraform workflows, automating repetitive tasks and reducing execution time.

Evaluation and Results

To assess the success of this deployment, the infrastructure was tested in a controlled environment simulating real production and development needs. The evaluation revealed:

  • Consistent Resource Deployment: Automation with Terraform ensured identical configurations across all environments.
  • Scalability: The hub-and-spoke architecture allowed the addition of new VPCs and subnets without reconfiguring the base infrastructure.
  • Reduction in Configuration Errors: Automated processes minimized human intervention, increasing deployment reliability.

Key Benefits

  • Centralized Resource Management: Centralized control of shared services within the Landing Zone.
  • Accelerated Deployment Cycles: Automation tools enabled faster provisioning and updates.
  • Enhanced Security and Governance: Compliance with AWS best practices provided a secure foundation for critical applications.
